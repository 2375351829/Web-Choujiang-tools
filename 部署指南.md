# 部署指南

## 1. 环境准备

### 1.1 系统要求

- **操作系统**：Linux、Windows或macOS
- **Node.js**：版本14.0或更高
- **npm**：版本6.0或更高
- **内存**：至少512MB RAM
- **存储空间**：至少100MB可用空间

### 1.2 依赖安装

在部署前，确保已安装以下软件：

- **Node.js**：从 [Node.js官网](https://nodejs.org/) 下载并安装
- **Git**：用于克隆代码仓库（可选）

## 2. 项目获取

### 2.1 克隆代码仓库

```bash
git clone https://github.com/2375351829/Web-Choujiang-tools.git
cd Web-Choujiang-tools
```

### 2.2 直接下载代码

如果没有Git，可以直接从GitHub下载代码压缩包，然后解压到本地目录。

## 3. 依赖安装

在项目根目录执行以下命令安装所有依赖：

```bash
npm install
```

## 4. 配置调整

### 4.1 服务器配置

根据实际部署环境，调整`server.js`中的服务器配置：

```javascript
const port = 3002; // 根据实际情况修改端口号
```

### 4.2 前端配置

如果需要修改前端API调用地址，调整相关配置。

## 5. 构建前端代码

在生产环境部署前，需要构建前端代码：

```bash
npm run build
```

构建完成后，前端代码会生成在`dist`目录中。

## 6. 部署方式

### 6.1 直接部署

#### 6.1.1 启动服务器

```bash
node server.js
```

#### 6.1.2 使用进程管理工具

为了确保服务稳定运行，建议使用PM2等进程管理工具：

1. 安装PM2：

```bash
npm install -g pm2
```

2. 启动应用：

```bash
npm start # 或 pm2 start server.js --name lottery-tool
```

3. 设置开机自启动：

```bash
pm run startup
```

### 6.2 Docker部署

#### 6.2.1 创建Dockerfile

在项目根目录创建`Dockerfile`文件：

```dockerfile
FROM node:16-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

RUN npm run build

EXPOSE 3002

CMD ["node", "server.js"]
```

#### 6.2.2 构建Docker镜像

```bash
docker build -t lottery-tool .
```

#### 6.2.3 运行Docker容器

```bash
docker run -d -p 3002:3002 --name lottery-tool lottery-tool
```

### 6.3 云服务器部署

#### 6.3.1 上传代码

使用FTP或SSH将代码上传到云服务器。

#### 6.3.2 安装依赖

在服务器上执行：

```bash
npm install
```

#### 6.3.3 构建前端代码

```bash
npm run build
```

#### 6.3.4 启动服务

使用PM2启动服务：

```bash
npm install -g pm2
npm start
```

## 10. 常见问题及解决方案

### 10.1 端口占用

**问题**：启动服务时提示端口已被占用

**解决方案**：
1. 查看端口占用情况：
   ```bash
   lsof -i :3002
   ```
2. 停止占用端口的进程或修改`server.js`中的端口号

### 10.2 依赖安装失败

**问题**：`npm install` 失败

**解决方案**：
1. 检查网络连接
2. 清除npm缓存：
   ```bash
   npm cache clean --force
   ```
3. 重新安装依赖

### 10.3 数据库连接失败

**问题**：启动服务时提示数据库连接失败

**解决方案**：
1. 确保`database.db`文件存在且有读写权限
2. 检查`db.js`中的数据库路径配置

### 10.4 文件上传失败

**问题**：上传文件时失败

**解决方案**：
1. 确保`uploads`目录存在且有写入权限
2. 检查文件大小是否超过限制
3. 检查文件类型是否被允许

## 13. 故障排查

### 13.1 查看日志

```bash
# 使用PM2查看日志
npm run logs

# 查看服务器日志
node server.js > server.log 2>&1
```

### 13.2 检查服务状态

```bash
# 检查服务是否运行
ps aux | grep node

# 检查端口是否开放
netstat -tuln | grep 3002
```

### 13.3 常见错误排查

| 错误信息 | 可能原因 | 解决方案 |
|---------|---------|--------|
| EADDRINUSE | 端口被占用 | 更换端口或停止占用端口的进程 |
| EACCES | 权限不足 | 确保文件和目录有正确的权限 |
| ENOTFOUND | 网络连接问题 | 检查网络连接和API地址配置 |
| SyntaxError | 代码语法错误 | 检查代码语法并修复 |
