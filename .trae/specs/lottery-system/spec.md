# 企业级通用网页抽奖系统 - 产品需求文档

## 概述
- **摘要**：开发一套企业级通用网页抽奖系统，包含管理员配置的后台管理端和观众参与的前端展示端，支持多种场景下的抽奖活动。
- **目的**：为企业年会、活动庆典等场景提供高度可配置的抽奖解决方案，确保公平性和良好的用户体验。
- **目标用户**：企业活动组织者（管理员）和参与抽奖的观众。

## 目标
- 构建完整的后台管理系统，支持人员管理、奖品管理、抽奖组合配置。
- 开发视觉效果丰富的前端展示端，支持多种抽奖动画模式。
- 实现后台与前端的实时通信，确保操作实时同步。
- 提供高度可配置的系统参数，适应不同场景需求。
- 确保抽奖算法的公平性和性能稳定性。

## 非目标（范围外）
- 不支持大规模并发用户（超过1000人同时参与）。
- 不提供用户注册和登录功能，仅通过后台导入人员名单。
- 不支持支付或交易功能。
- 不支持多语言国际化。

## 背景与上下文
- 企业活动中抽奖环节是常见的互动方式，需要一个灵活、可配置的系统来满足不同场景需求。
- 传统抽奖系统往往缺乏灵活性和视觉效果，难以适应现代企业活动的需求。
- 轻量级部署需求，无需复杂的数据库和服务器配置。

## 功能需求
- **FR-1**：人员管理 - 支持Excel批量导入名单，手动增删改查，设置中奖状态，支持黑名单功能。
- **FR-2**：奖品管理 - 自定义奖品名称、图片、数量、中奖概率权重、单次抽取人数。
- **FR-3**：抽奖组合 - 支持创建不同的抽奖场次，灵活组合奖品与参与人员池。
- **FR-4**：控制台 - 提供开始、暂停、重置、显示中奖名单等操作按钮，操作需实时映射到前端。
- **FR-5**：前端视觉效果 - 全屏沉浸式背景，支持自定义背景图/视频。
- **FR-6**：抽奖动画 - 支持滚动态和揭晓态，带有特效和音效。
- **FR-7**：多模式展示 - 支持九宫格模式、列表滚动模式、3D球体滚动模式。
- **FR-8**：高度自定义 - 后台可修改前端展示的标题、背景图、滚动速度、配色方案。
- **FR-9**：响应式布局 - 前端展示端适配投影仪大屏（16:9或超宽屏）。

## 非功能需求
- **NFR-1**：性能优化 - 抽奖过程需保证60FPS流畅度，无卡顿。
- **NFR-2**：可靠性 - 抽奖算法需保证公平性，结果可追溯。
- **NFR-3**：可部署性 - 支持轻量级部署，使用SQLite或JSON文件存储。
- **NFR-4**：实时性 - 后台操作需实时同步到前端展示。

## 约束
- **技术**：
  - 后端：Node.js + Express/Koa
  - 数据库：SQLite或JSON文件存储
  - 前端：Vue 3 + Vite
  - UI框架：Element Plus（后台） + 自定义CSS3动画（前端）
  - 通信：Axios + Socket.io（推荐）
- **业务**：无特殊业务约束
- **依赖**：需要Excel解析库（如xlsx）用于人员导入。

## 假设
- 人员名单数据量不超过10000条。
- 抽奖活动时间不超过4小时。
- 网络环境稳定，支持WebSocket连接。

## 验收标准

### AC-1：人员管理功能
- **Given**：管理员登录后台管理端
- **When**：上传Excel文件导入人员名单
- **Then**：系统成功解析并存储人员信息，支持手动编辑和状态管理
- **Verification**：`programmatic`

### AC-2：奖品管理功能
- **Given**：管理员在后台配置奖品
- **When**：设置奖品名称、数量、概率权重等参数
- **Then**：系统正确存储奖品信息，并在抽奖时按配置执行
- **Verification**：`programmatic`

### AC-3：抽奖组合配置
- **Given**：管理员创建抽奖场次
- **When**：选择参与人员池和奖品
- **Then**：系统成功创建抽奖组合，可随时修改
- **Verification**：`programmatic`

### AC-4：控制台操作
- **Given**：管理员在控制台操作
- **When**：点击开始、暂停、重置等按钮
- **Then**：前端展示端实时响应操作，显示相应状态
- **Verification**：`programmatic`

### AC-5：前端视觉效果
- **Given**：观众访问前端展示端
- **When**：抽奖活动进行时
- **Then**：展示全屏沉浸式背景，动画流畅，视觉效果良好
- **Verification**：`human-judgment`

### AC-6：抽奖动画效果
- **Given**：抽奖活动进行中
- **When**：管理员点击开始抽奖
- **Then**：前端显示滚动态效果，结束时显示揭晓态，伴随特效和音效
- **Verification**：`human-judgment`

### AC-7：多模式展示
- **Given**：管理员在后台配置展示模式
- **When**：前端展示端加载
- **Then**：按照配置显示相应的展示模式（九宫格、列表、3D球体）
- **Verification**：`human-judgment`

### AC-8：自定义配置
- **Given**：管理员在后台修改前端配置
- **When**：保存配置后
- **Then**：前端展示端实时更新显示效果
- **Verification**：`programmatic`

### AC-9：响应式布局
- **Given**：前端展示端在不同屏幕尺寸下
- **When**：访问页面
- **Then**：页面自适应不同屏幕尺寸，特别是投影仪大屏
- **Verification**：`human-judgment`

### AC-10：性能要求
- **Given**：抽奖活动进行中
- **When**：系统执行抽奖动画
- **Then**：动画流畅度达到60FPS，无卡顿现象
- **Verification**：`programmatic`
