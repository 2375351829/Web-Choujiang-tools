# 配置说明文档

## 1. 项目配置概述

本项目是一个基于Node.js和Vue.js的网页抽奖工具，包含前端和后端两部分。本文档详细说明项目的配置项、配置方法和注意事项。

## 2. 后端配置

### 2.1 服务器配置

在`server.js`文件中，服务器配置主要包括：

| 配置项 | 说明 | 默认值 | 修改方法 |
|-------|------|-------|---------|
| 端口号 | 服务器监听的端口 | 3002 | 修改`server.js`中的`port`变量 |
| 文件上传目录 | 上传文件的存储目录 | uploads/ | 修改`multer`配置中的`dest`属性 |
| 文件大小限制 | 上传文件的最大大小 | 10MB | 修改`multer`配置中的`limits.fileSize`属性 |
| 允许的文件类型 | 允许上传的文件类型 | 图片和Excel文件 | 修改`multer`配置中的`fileFilter`函数 |

### 2.2 数据库配置

项目使用SQLite数据库，配置主要在`db.js`文件中：

| 配置项 | 说明 | 默认值 | 修改方法 |
|-------|------|-------|---------|
| 数据库文件路径 | SQLite数据库文件的存储位置 | database.db | 修改`db.js`中的数据库连接路径 |
| 数据库表结构 | 数据库表的设计和索引 | 见数据库表结构部分 | 修改`initDatabase`函数中的SQL语句 |

### 2.3 路由配置

API路由配置在`server.js`文件中，通过以下方式注册：

```javascript
// 导入路由
const participantsRouter = require('./routes/participants');
const prizesRouter = require('./routes/prizes');
const sessionsRouter = require('./routes/sessions');
const consoleRouter = require('./routes/console');

// 注册路由
app.use('/api/participants', participantsRouter);
app.use('/api/prizes', prizesRouter);
app.use('/api/sessions', sessionsRouter);
app.use('/api/console', consoleRouter);
```

## 3. 前端配置

### 3.1 Vite配置

Vite配置文件为`vite.config.js`，主要配置项包括：

| 配置项 | 说明 | 默认值 | 修改方法 |
|-------|------|-------|---------|
| 开发服务器端口 | 前端开发服务器端口 | 5173 | 修改`vite.config.js`中的`server.port`配置 |
| 代理配置 | 开发环境API代理 | 无 | 添加`server.proxy`配置 |
| 构建配置 | 生产环境构建配置 | 无 | 添加`build`配置 |

### 3.2 前端API调用配置

前端通过API调用与后端交互，默认情况下，前端会向当前域名的`/api`路径发送请求。如果需要修改API基础路径，可以在前端代码中修改。


## 5. 环境配置

### 5.1 开发环境

开发环境下，前端使用Vite开发服务器，后端使用Express服务器。

启动开发服务器：

```bash
npm run dev
```

### 5.2 生产环境

生产环境下，需要先构建前端代码，然后启动后端服务器。

构建前端代码：

```bash
npm run build
```

启动后端服务器：

```bash
node server.js
```

## 6. 安全配置

### 6.1 文件上传安全

项目对文件上传进行了以下安全配置：

- 限制文件大小（最大10MB）
- 限制文件类型（仅允许图片和Excel文件）
- 上传文件存储在`uploads`目录，通过Express静态文件服务访问

### 6.2 API安全

项目目前没有实现API认证，在生产环境部署时，建议添加以下安全措施：

- 添加API认证机制（如JWT）
- 实现请求频率限制
- 添加CORS配置
- 对敏感操作添加权限控制

## 7. 配置示例

### 7.1 服务器端口修改示例

修改`server.js`文件：

```javascript
const port = 8080; // 修改为8080端口

// 启动服务器
server.listen(port, () => {
  console.log(`服务器运行在 http://localhost:${port}`);
});
```

### 7.2 Vite代理配置示例

修改`vite.config.js`文件：

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3002',
        changeOrigin: true
      }
    }
  }
})
```

## 8. 注意事项

1. **数据库备份**：定期备份`database.db`文件，以防数据丢失
2. **文件上传**：确保`uploads`目录存在且有写入权限
3. **端口占用**：确保配置的端口未被其他服务占用
4. **依赖版本**：使用指定版本的依赖，避免版本冲突
5. **生产环境**：在生产环境中，建议使用PM2等进程管理工具来管理Node.js应用
6. **安全措施**：在生产环境中，添加适当的安全措施，如API认证、HTTPS等
