## 📊 表结构详细说明

### 1. participants (参与者表)

存储所有抽奖参与者的基本信息。

```sql
CREATE TABLE participants (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT,
    phone TEXT,
    avatar TEXT DEFAULT '',
    status TEXT DEFAULT '未中奖',
    is_blacklisted INTEGER DEFAULT 0,
    weight INTEGER DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 字段说明

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | INTEGER | PK, AUTO | 自增 | 主键，唯一标识 |
| name | TEXT | NOT NULL | - | 参与者姓名，必填 |
| email | TEXT | - | - | 邮箱地址，可为空 |
| phone | TEXT | - | - | 电话号码，可为空 |
| avatar | TEXT | - | '' | 头像URL路径 |
| status | TEXT | - | '未中奖' | 参与状态：已中奖/未中奖 |
| is_blacklisted | INTEGER | - | 0 | 黑名单标志：0=正常, 1=黑名单 |
| weight | INTEGER | - | 1 | 中奖权重，用于加权抽奖 |
| created_at | DATETIME | - | NOW | 创建时间戳 |

#### 索引

```sql
CREATE INDEX idx_participants_status ON participants(status);
CREATE INDEX idx_participants_blacklisted ON participants(is_blacklisted);
```

#### 示例数据

| id | name | email | phone | status | is_blacklisted | weight |
|----|------|-------|-------|--------|----------------|--------|
| 1 | 张三 | zhangsan@example.com | 13800138000 | 未中奖 | 0 | 1 |
| 2 | 李四 | lisi@example.com | 13900139000 | 已中奖 | 0 | 1 |
| 3 | 王五 | wangwu@example.com | 13700137000 | 未中奖 | 0 | 2 |
| 4 | 赵六 | zhaoliu@example.com | 13600136000 | 未中奖 | 1 | 1 |

#### 业务规则

1. **姓名唯一性**: 同一场次内，同一姓名不能重复参与
2. **状态同步**: 通过数据库触发器自动同步
3. **权重设计**: weight字段支持差异化中奖概率

### 2. prizes (奖品表)

存储所有奖品的信息。

```sql
CREATE TABLE prizes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT,
    image TEXT,
    weight INTEGER DEFAULT 1,
    draw_count INTEGER DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 字段说明

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | INTEGER | PK, AUTO | 自增 | 主键，唯一标识 |
| name | TEXT | NOT NULL | - | 奖品名称，必填 |
| description | TEXT | - | - | 奖品详细描述 |
| image | TEXT | - | - | 奖品图片URL路径 |
| weight | INTEGER | - | 1 | 奖品权重 |
| draw_count | INTEGER | - | 1 | 可抽取次数 |
| created_at | DATETIME | - | NOW | 创建时间戳 |

#### 索引

```sql
-- 无默认索引，如有需要可添加
-- CREATE INDEX idx_prizes_name ON prizes(name);
```

#### 示例数据

| id | name | description | image | weight | draw_count |
|----|------|-------------|-------|--------|------------|
| 1 | 一等奖-iPhone | iPhone 15 Pro 256GB | /uploads/iphone.png | 1 | 1 |
| 2 | 二等奖-AirPods | AirPods Pro 2 | /uploads/airpods.png | 1 | 3 |
| 3 | 三等奖-耳机 | 小米蓝牙耳机 | /uploads/earphones.png | 1 | 10 |

#### 业务规则

1. **数量限制**: draw_count表示该奖品总共可被抽中的次数
2. **库存追踪**: 每抽中一次，库存减一

### 3. lottery_sessions (抽奖场次表)

存储抽奖场次的信息。

```sql
CREATE TABLE lottery_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT,
    status TEXT DEFAULT '未开始',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 字段说明

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | INTEGER | PK, AUTO | 自增 | 主键，唯一标识 |
| name | TEXT | NOT NULL | - | 场次名称，必填 |
| description | TEXT | - | - | 场次描述 |
| status | TEXT | - | '未开始' | 场次状态 |
| created_at | DATETIME | - | NOW | 创建时间 |
| updated_at | DATETIME | - | NOW | 最后更新时间 |

#### 状态值

| 状态值 | 说明 |
|--------|------|
| 未开始 | 场次刚创建，未进行抽奖 |
| 进行中 | 正在抽奖中 |
| 已结束 | 抽奖活动已完成 |

#### 示例数据

### 4. session_participants (场次-参与者关联表)

存储场次与参与者的多对多关系。

```sql
CREATE TABLE session_participants (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    participant_id INTEGER NOT NULL,
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id),
    FOREIGN KEY (participant_id) REFERENCES participants(id)
);
```

#### 字段说明

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PK, AUTO | 主键 |
| session_id | INTEGER | NOT NULL, FK | 关联的场次ID |
| participant_id | INTEGER | NOT NULL, FK | 关联的参与者ID |

#### 索引

```sql
CREATE UNIQUE INDEX idx_session_participant ON session_participants(session_id, participant_id);
```

#### ER图

```
lottery_sessions ──┬──< session_participants >─── participants
     (1)          │              │                   (1)
                  │              │
                  └──────────────┘
                    多对多关系
```

### 5. session_prizes (场次-奖品关联表)

存储场次与奖品的配置关系。

```sql
CREATE TABLE session_prizes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    prize_id INTEGER NOT NULL,
    quantity INTEGER DEFAULT 1,
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id),
    FOREIGN KEY (prize_id) REFERENCES prizes(id)
);
```

#### 字段说明

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | INTEGER | PK, AUTO | 自增 | 主键 |
| session_id | INTEGER | NOT NULL, FK | - | 关联的场次ID |
| prize_id | INTEGER | NOT NULL, FK | - | 关联的奖品ID |
| quantity | INTEGER | - | 1 | 该场次中此奖品的数量 |

#### 索引

```sql
CREATE UNIQUE INDEX idx_session_prize ON session_prizes(session_id, prize_id);
```

### 6. session_results (抽奖结果表)

存储每次抽奖的中奖结果。

```sql
CREATE TABLE session_results (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    participant_id INTEGER NOT NULL,
    prize_id INTEGER NOT NULL,
    drawn_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id),
    FOREIGN KEY (participant_id) REFERENCES participants(id),
    FOREIGN KEY (prize_id) REFERENCES prizes(id)
);
```

#### 字段说明

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | INTEGER | PK, AUTO | 自增 | 主键 |
| session_id | INTEGER | NOT NULL, FK | - | 关联的场次ID |
| participant_id | INTEGER | NOT NULL, FK | - | 中奖的参与者ID |
| prize_id | INTEGER | NOT NULL, FK | - | 获得的奖品ID |
| drawn_at | DATETIME | - | NOW | 中奖时间 |

#### 索引

```sql
CREATE INDEX idx_results_session ON session_results(session_id);
CREATE INDEX idx_results_participant ON session_results(participant_id);
CREATE INDEX idx_results_prize ON session_results(prize_id);
```

### 7. lottery_config (抽奖配置表)

存储系统的全局配置。

```sql
CREATE TABLE lottery_config (
    id INTEGER PRIMARY KEY DEFAULT 1,
    background_type TEXT DEFAULT 'gradient',
    background_image TEXT DEFAULT '',
    background_color TEXT DEFAULT '#0f172a',
    gradient_start TEXT DEFAULT '#667eea',
    gradient_end TEXT DEFAULT '#764ba2',
    gradient_degree INTEGER DEFAULT 135,
    show_participants INTEGER DEFAULT 1,
    show_prizes INTEGER DEFAULT 1,
    show_winner INTEGER DEFAULT 1,
    carousel_speed INTEGER DEFAULT 3000,
    title TEXT DEFAULT '抽奖活动',
    animation_type TEXT DEFAULT 'grid',
    current_session_id INTEGER
);
```

#### 字段说明

| 字段名 | 数据类型 | 默认值 | 说明 |
|--------|----------|--------|------|
| id | INTEGER | 1 | 主键（固定为1） |
| background_type | TEXT | 'gradient' | 背景类型：gradient/image/color |
| background_image | TEXT | '' | 背景图片URL |
| background_color | TEXT | '#0f172a' | 单色背景颜色 |
| gradient_start | TEXT | '#667eea' | 渐变起始色 |
| gradient_end | TEXT | '#764ba2' | 渐变终止色 |
| gradient_degree | INTEGER | 135 | 渐变角度(0-360) |
| show_participants | INTEGER | 1 | 是否显示参与者区 |
| show_prizes | INTEGER | 1 | 是否显示奖品区 |
| show_winner | INTEGER | 1 | 是否显示中奖区 |
| carousel_speed | INTEGER | 3000 | 轮播速度(ms) |
| title | TEXT | '抽奖活动' | 展示页面标题 |
| animation_type | TEXT | 'grid' | 动画类型：grid/sphere/list |
| current_session_id | INTEGER | - | 当前活跃场次ID |

#### 配置选项

**background_type 可选值**:
- `gradient`: 渐变背景
- `color`: 纯色背景
- `image`: 图片背景

**animation_type 可选值**:
- `grid`: 网格动画（3x3方格）
- `sphere`: 球体动画（3D效果）
- `list`: 列表动画（纵向滚动）

## 🔧 数据库触发器

### 1. 状态同步触发器

```sql
-- 插入抽奖结果时自动更新参与者状态
CREATE TRIGGER trg_participant_status_sync
AFTER INSERT ON session_results
FOR EACH ROW
BEGIN
    UPDATE participants
    SET status = '已中奖'
    WHERE id = NEW.participant_id;
END;

-- 删除抽奖结果时自动回退参与者状态
CREATE TRIGGER trg_participant_result_delete
AFTER DELETE ON session_results
FOR EACH ROW
BEGIN
    UPDATE participants
    SET status = '未中奖'
    WHERE id = OLD.participant_id
    AND id NOT IN (SELECT participant_id FROM session_results);
END;
```

### 2. 时间戳自动更新

```sql
-- 更新场次时自动更新时间戳
CREATE TRIGGER trg_session_updated_at
AFTER UPDATE ON lottery_sessions
FOR EACH ROW
BEGIN
    UPDATE lottery_sessions
    SET updated_at = CURRENT_TIMESTAMP
    WHERE id = NEW.id;
END;
```

## 📈 性能优化

### 1. 索引策略

| 表名 | 索引名 | 字段 | 用途 |
|------|--------|------|------|
| participants | idx_participants_status | status | 按状态筛选 |
| participants | idx_participants_blacklisted | is_blacklisted | 黑名单过滤 |
| session_participants | idx_session_participant | (session_id, participant_id) | 唯一约束+查询加速 |
| session_prizes | idx_session_prize | (session_id, prize_id) | 唯一约束+查询加速 |
| session_results | idx_results_session | session_id | 结果查询 |
| session_results | idx_results_participant | participant_id | 个人中奖查询 |
| session_results | idx_results_prize | prize_id | 奖品统计 |

### 2. 查询优化示例

**获取场次的可用参与者**（排除已中奖和黑名单）:
```sql
SELECT p.*
FROM participants p
JOIN session_participants sp ON p.id = sp.participant_id
WHERE sp.session_id = ?
  AND p.is_blacklisted = 0
  AND p.id NOT IN (
    SELECT participant_id
    FROM session_results
    WHERE session_id = ? AND prize_id = ?
  );
```

**统计奖品剩余数量**:
```sql
SELECT
    sp.prize_id,
    sp.quantity - COUNT(sr.id) as remaining
FROM session_prizes sp
LEFT JOIN session_results sr ON sp.session_id = sr.session_id AND sp.prize_id = sr.prize_id
WHERE sp.session_id = ?
GROUP BY sp.prize_id;
```

## 🔒 数据完整性

### 1. 外键约束

所有关联表都设置了外键约束：
- `session_participants.session_id` → `lottery_sessions.id`
- `session_participants.participant_id` → `participants.id`
- `session_prizes.session_id` → `lottery_sessions.id`
- `session_prizes.prize_id` → `prizes.id`
- `session_results.session_id` → `lottery_sessions.id`
- `session_results.participant_id` → `participants.id`
- `session_results.prize_id` → `prizes.id`

### 2. 约束验证

删除有关联数据的记录时会触发外键约束错误，除非使用 `ON DELETE CASCADE`。

### 3. 业务层保护

除数据库约束外，业务逻辑层也进行了额外校验：
- API层检查重复创建
- 抽奖前检查库存
- 抽奖时检查参与者资格

## 💾 数据库维护

### 1. 备份

```bash
# 手动备份
cp database.db database_backup_$(date +%Y%m%d).db

# 或使用 SQLite VACUUM 优化
sqlite3 database.db "VACUUM;"
```

### 2. 查看数据库状态

```sql
-- 查看表大小
SELECT
    name,
    tbl_name,
    SUM(cnt) as total_rows
FROM (
    SELECT count(*) as cnt, 'participants' as name, 'participants' as tbl_name FROM participants
    UNION ALL
    SELECT count(*), 'prizes', 'prizes' FROM prizes
    UNION ALL
    SELECT count(*), 'lottery_sessions', 'lottery_sessions' FROM lottery_sessions
    UNION ALL
    SELECT count(*), 'session_participants', 'session_participants' FROM session_participants
    UNION ALL
    SELECT count(*), 'session_prizes', 'session_prizes' FROM session_prizes
    UNION ALL
    SELECT count(*), 'session_results', 'session_results' FROM session_results
)
GROUP BY name;
```
