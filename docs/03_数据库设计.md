## 3. 表结构详细设计

### 3.1 participants (参与者表)

存储所有参与者信息，包括联系方式、中奖状态和黑名单标记。

```sql
CREATE TABLE participants (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT DEFAULT '',
    phone TEXT DEFAULT '',
    avatar TEXT DEFAULT '',
    status TEXT DEFAULT '未中奖' CHECK(status IN ('未中奖', '已中奖')),
    is_blacklisted INTEGER DEFAULT 0 CHECK(is_blacklisted IN (0, 1)),
    weight INTEGER DEFAULT 1 CHECK(weight >= 0),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**字段说明：**

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增 | 参与者唯一标识 |
| name | TEXT | NOT NULL | - | 参与者姓名 |
| email | TEXT | - | '' | 邮箱地址 |
| phone | TEXT | - | '' | 手机号码 |
| avatar | TEXT | - | '' | 头像URL或Base64 |
| status | TEXT | CHECK | '未中奖' | 中奖状态：未中奖 / 已中奖 |
| is_blacklisted | INTEGER | CHECK | 0 | 是否被拉黑：0-否, 1-是 |
| weight | INTEGER | CHECK | 1 | 抽奖权重，0表示不参与抽奖 |
| created_at | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |

**索引：**

```sql
CREATE INDEX idx_participants_status ON participants(status);
CREATE INDEX idx_participants_blacklisted ON participants(is_blacklisted);
CREATE UNIQUE INDEX idx_participants_name_email ON participants(name, email);
```

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| idx_participants_status | status | 普通索引 | 按中奖状态筛选参与者 |
| idx_participants_blacklisted | is_blacklisted | 普通索引 | 快速过滤黑名单用户 |
| idx_participants_name_email | name, email | 唯一索引 | 防止同名同邮箱重复导入 |

**示例数据：**

```sql
INSERT INTO participants (name, email, phone, avatar, status, is_blacklisted, weight) VALUES
('张三', 'zhangsan@example.com', '13800138001', '', '未中奖', 0, 1),
('李四', 'lisi@example.com', '13800138002', '', '未中奖', 0, 2),
('王五', 'wangwu@example.com', '13800138003', '', '已中奖', 0, 1),
('赵六', 'zhaoliu@example.com', '13800138004', '', '未中奖', 1, 0);
```

**字段说明：**

| 字段名 | 类型 | 约束 | 默认值 | 说明 |

| name | TEXT | NOT NULL | - | 奖品名称 |

### 3.3 lottery_sessions (抽奖场次表)

存储抽奖场次信息，每个场次独立管理参与者和奖品。

```sql
CREATE TABLE lottery_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT DEFAULT '',
    status TEXT DEFAULT '未开始' CHECK(status IN ('未开始', '进行中', '已结束')),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**字段说明：**

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增 | 场次唯一标识 |
| name | TEXT | NOT NULL | - | 场次名称 |
| description | TEXT | - | '' | 场次描述 |
| status | TEXT | CHECK | '未开始' | 场次状态 |
| created_at | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | CURRENT_TIMESTAMP | 最后更新时间 |

**状态值说明：**

| 状态值 | 说明 |
|--------|------|
| 未开始 | 场次已创建但尚未开始抽奖，可继续添加参与者和奖品 |
| 进行中 | 场次正在抽奖中，可执行抽奖操作 |
| 已结束 | 场次已结束，不可再进行抽奖操作 |

**示例数据：**

```sql
INSERT INTO lottery_sessions (name, description, status) VALUES
('2026年年会抽奖', '公司年度抽奖活动', '进行中'),
('新人入职抽奖', '欢迎新员工抽奖', '未开始');
```

### 3.4 session_participants (场次-参与者关联表)

管理场次与参与者的多对多关系。

```sql
CREATE TABLE session_participants (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    participant_id INTEGER NOT NULL,
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id) ON DELETE CASCADE,
    FOREIGN KEY (participant_id) REFERENCES participants(id) ON DELETE CASCADE,
    UNIQUE(session_id, participant_id)
);
```

**字段说明：**

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增 | 关联记录唯一标识 |
| session_id | INTEGER | NOT NULL, FK | - | 关联的场次ID |
| participant_id | INTEGER | NOT NULL, FK | - | 关联的参与者ID |

**约束说明：**

- `UNIQUE(session_id, participant_id)`：同一参与者在同一场次中只能出现一次
- `ON DELETE CASCADE`：删除场次或参与者时，自动删除关联记录

### 3.5 session_prizes (场次-奖品关联表)

管理场次与奖品的多对多关系，并记录每个奖品在该场次的数量。

```sql
CREATE TABLE session_prizes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    prize_id INTEGER NOT NULL,
    quantity INTEGER DEFAULT 1 CHECK(quantity > 0),
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id) ON DELETE CASCADE,
    FOREIGN KEY (prize_id) REFERENCES prizes(id) ON DELETE CASCADE,
    UNIQUE(session_id, prize_id)
);
```

**字段说明：**

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增 | 关联记录唯一标识 |
| session_id | INTEGER | NOT NULL, FK | - | 关联的场次ID |
| prize_id | INTEGER | NOT NULL, FK | - | 关联的奖品ID |
| quantity | INTEGER | CHECK | 1 | 该奖品在本场次的数量 |

**约束说明：**

- `UNIQUE(session_id, prize_id)`：同一奖品在同一场次中只能配置一次
- `quantity > 0`：奖品数量必须大于0

### 3.6 session_results (抽奖结果表)

记录每场抽奖的结果，包括中奖者、对应奖品和抽奖时间。

```sql
CREATE TABLE session_results (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    participant_id INTEGER NOT NULL,
    prize_id INTEGER NOT NULL,
    drawn_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id) ON DELETE CASCADE,
    FOREIGN KEY (participant_id) REFERENCES participants(id) ON DELETE CASCADE,
    FOREIGN KEY (prize_id) REFERENCES prizes(id) ON DELETE CASCADE,
    UNIQUE(session_id, participant_id, prize_id)
);
```

**字段说明：**

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增 | 结果记录唯一标识 |
| session_id | INTEGER | NOT NULL, FK | - | 所属场次ID |
| participant_id | INTEGER | NOT NULL, FK | - | 中奖参与者ID |
| prize_id | INTEGER | NOT NULL, FK | - | 中奖奖品ID |
| drawn_at | DATETIME | - | CURRENT_TIMESTAMP | 抽奖时间 |

**约束说明：**

- `UNIQUE(session_id, participant_id, prize_id)`：同一参与者在同一场次中对同一奖品只能中奖一次
- `ON DELETE CASCADE`：删除场次、参与者或奖品时，自动删除相关结果

**索引：**

```sql
CREATE INDEX idx_results_session ON session_results(session_id);
CREATE INDEX idx_results_participant ON session_results(participant_id);
CREATE INDEX idx_results_prize ON session_results(prize_id);
```

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| idx_results_session | session_id | 普通索引 | 按场次查询抽奖结果 |
| idx_results_participant | participant_id | 普通索引 | 按参与者查询中奖记录 |
| idx_results_prize | prize_id | 普通索引 | 按奖品查询中奖记录 |

## 4. 触发器设计

### 4.1 参与者状态同步触发器

当抽奖结果变化时，自动同步参与者的中奖状态。

```sql
-- 插入结果时自动标记为已中奖
CREATE TRIGGER trg_participant_status_sync
AFTER INSERT ON session_results
FOR EACH ROW
BEGIN
    UPDATE participants SET status = '已中奖'
    WHERE id = NEW.participant_id;
END;

-- 删除结果时检查是否还有其他中奖记录
CREATE TRIGGER trg_participant_result_delete
AFTER DELETE ON session_results
FOR EACH ROW
BEGIN
    UPDATE participants SET status = '未中奖'
    WHERE id = OLD.participant_id
    AND id NOT IN (SELECT participant_id FROM session_results);
END;
```

**触发器说明：**

| 触发器名 | 触发时机 | 作用 |
|----------|----------|------|
| trg_participant_status_sync | INSERT session_results 后 | 将中奖参与者状态自动更新为"已中奖" |
| trg_participant_result_delete | DELETE session_results 后 | 删除中奖记录后，若该参与者无其他中奖记录则恢复为"未中奖" |

---

### 4.2 场次时间戳更新触发器

自动更新场次的 `updated_at` 字段。

```sql
CREATE TRIGGER trg_session_updated_at
AFTER UPDATE ON lottery_sessions
FOR EACH ROW
BEGIN
    UPDATE lottery_sessions SET updated_at = CURRENT_TIMESTAMP
    WHERE id = NEW.id;
END;
```

**触发器说明：**

| 触发器名 | 触发时机 | 作用 |
|----------|----------|------|
| trg_session_updated_at | UPDATE lottery_sessions 后 | 自动更新场次的 updated_at 为当前时间戳 |

## 6. 常用查询

### 6.1 获取场次可用参与者

查询指定场次中未被拉黑且尚未获得指定奖品的参与者列表。

```sql
SELECT p.* FROM participants p
JOIN session_participants sp ON p.id = sp.participant_id
WHERE sp.session_id = ? AND p.is_blacklisted = 0
AND p.id NOT IN (
    SELECT participant_id FROM session_results
    WHERE session_id = ? AND prize_id = ?
);
```

**参数说明：**
- 第1个 `?`：场次ID
- 第2个 `?`：场次ID（同上）
- 第3个 `?`：奖品ID

---

### 6.2 统计奖品剩余数量

查询指定场次中各奖品的剩余可抽数量。

```sql
SELECT sp.prize_id, sp.quantity - COUNT(sr.id) as remaining
FROM session_prizes sp
LEFT JOIN session_results sr ON sp.session_id = sr.session_id AND sp.prize_id = sr.prize_id
WHERE sp.session_id = ?
GROUP BY sp.prize_id;
```

**参数说明：**
- `?`：场次ID

**返回字段：**
- `prize_id`：奖品ID
- `remaining`：剩余可抽数量

---

### 6.3 获取场次完整结果

查询指定场次的所有抽奖结果，包含参与者和奖品详细信息。

```sql
SELECT sr.drawn_at, p.name as participant_name, p.email, p.phone, pr.name as prize_name
FROM session_results sr
JOIN participants p ON sr.participant_id = p.id
JOIN prizes pr ON sr.prize_id = pr.id
WHERE sr.session_id = ?
ORDER BY sr.drawn_at DESC;
```

**参数说明：**
- `?`：场次ID

**返回字段：**
- `drawn_at`：抽奖时间
- `participant_name`：中奖者姓名
- `email`：中奖者邮箱
- `phone`：中奖者手机
- `prize_name`：奖品名称

## 7. 数据库初始化脚本

```sql
-- ============================================
-- 抽奖系统数据库初始化脚本
-- 数据库: SQLite 3
-- 版本: v1.0.0
-- ============================================

-- 启用WAL模式和外键约束
PRAGMA journal_mode = WAL;
PRAGMA foreign_keys = ON;

-- ============================================
-- 创建表
-- ============================================

-- 参与者表
CREATE TABLE IF NOT EXISTS participants (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT DEFAULT '',
    phone TEXT DEFAULT '',
    avatar TEXT DEFAULT '',
    status TEXT DEFAULT '未中奖' CHECK(status IN ('未中奖', '已中奖')),
    is_blacklisted INTEGER DEFAULT 0 CHECK(is_blacklisted IN (0, 1)),
    weight INTEGER DEFAULT 1 CHECK(weight >= 0),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 奖品表
CREATE TABLE IF NOT EXISTS prizes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT DEFAULT '',
    image TEXT DEFAULT '',
    weight INTEGER DEFAULT 1,
    draw_count INTEGER DEFAULT 1 CHECK(draw_count > 0),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 抽奖场次表
CREATE TABLE IF NOT EXISTS lottery_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    description TEXT DEFAULT '',
    status TEXT DEFAULT '未开始' CHECK(status IN ('未开始', '进行中', '已结束')),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 场次-参与者关联表
CREATE TABLE IF NOT EXISTS session_participants (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER NOT NULL,
    participant_id INTEGER NOT NULL,
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id) ON DELETE CASCADE,

    UNIQUE(session_id, participant_id)

-- 场次-奖品关联表

    quantity INTEGER DEFAULT 1 CHECK(quantity > 0),
    FOREIGN KEY (session_id) REFERENCES lottery_sessions(id) ON DELETE CASCADE,
    FOREIGN KEY (prize_id) REFERENCES prizes(id) ON DELETE CASCADE,
    UNIQUE(session_id, prize_id)
);

-- 抽奖结果表
CREATE TABLE IF NOT EXISTS session_results (

    UNIQUE(session_id, participant_id, prize_id)

-- 系统配置表
CREATE TABLE IF NOT EXISTS lottery_config (
    id INTEGER PRIMARY KEY DEFAULT 1,
    background_type TEXT DEFAULT 'gradient' CHECK(background_type IN ('gradient', 'color', 'image')),
    background_image TEXT DEFAULT '',
    background_color TEXT DEFAULT '#0f172a',
    gradient_start TEXT DEFAULT '#667eea',
    gradient_end TEXT DEFAULT '#764ba2',
    gradient_degree INTEGER DEFAULT 135 CHECK(gradient_degree BETWEEN 0 AND 360),
    show_participants INTEGER DEFAULT 1,
    show_prizes INTEGER DEFAULT 1,
    show_winner INTEGER DEFAULT 1,
    carousel_speed INTEGER DEFAULT 3000 CHECK(carousel_speed BETWEEN 500 AND 10000),
    title TEXT DEFAULT '抽奖活动',
    animation_type TEXT DEFAULT 'grid' CHECK(animation_type IN ('grid', 'sphere', 'list')),
    current_session_id INTEGER,
    theme TEXT DEFAULT 'dark' CHECK(theme IN ('dark', 'light', 'ocean', 'forest', 'sunset', 'purple'))
);

-- ============================================
-- 创建索引
-- ============================================

CREATE INDEX IF NOT EXISTS idx_participants_status ON participants(status);
CREATE INDEX IF NOT EXISTS idx_participants_blacklisted ON participants(is_blacklisted);
CREATE UNIQUE INDEX IF NOT EXISTS idx_participants_name_email ON participants(name, email);
CREATE INDEX IF NOT EXISTS idx_results_session ON session_results(session_id);
CREATE INDEX IF NOT EXISTS idx_results_participant ON session_results(participant_id);
CREATE INDEX IF NOT EXISTS idx_results_prize ON session_results(prize_id);

-- ============================================
-- 创建触发器
-- ============================================

-- 插入结果时自动标记为已中奖
CREATE TRIGGER IF NOT EXISTS trg_participant_status_sync
AFTER INSERT ON session_results
FOR EACH ROW
BEGIN
    UPDATE participants SET status = '已中奖'
    WHERE id = NEW.participant_id;
END;

-- 删除结果时检查是否还有其他中奖记录
CREATE TRIGGER IF NOT EXISTS trg_participant_result_delete
AFTER DELETE ON session_results
FOR EACH ROW
BEGIN
    UPDATE participants SET status = '未中奖'
    WHERE id = OLD.participant_id
    AND id NOT IN (SELECT participant_id FROM session_results);
END;

-- 场次更新时自动更新时间戳
CREATE TRIGGER IF NOT EXISTS trg_session_updated_at
AFTER UPDATE ON lottery_sessions
FOR EACH ROW
BEGIN
    UPDATE lottery_sessions SET updated_at = CURRENT_TIMESTAMP
    WHERE id = NEW.id;
END;

-- ============================================
-- 插入默认配置
-- ============================================

INSERT OR IGNORE INTO lottery_config (id) VALUES (1);
```

## 8. 数据库维护

### 8.1 备份策略

SQLite 数据库为单文件存储，备份方式简单高效。

**在线备份（推荐）：**

```sql
-- 使用SQLite内置备份API（通过命令行）
.backup server/data/lottery_backup_20260403.db
```

**文件级备份：**

```bash
# 直接复制数据库文件（需确保无写入操作）
cp server/data/lottery.db server/data/lottery_backup_20260403.db

# 同时复制WAL和SHM文件
cp server/data/lottery.db-wal server/data/lottery_backup_20260403.db-wal
cp server/data/lottery.db-shm server/data/lottery_backup_20260403.db-shm
```

**备份建议：**

| 策略 | 说明 |
|------|------|
| 备份频率 | 每日自动备份，重要活动前手动备份 |
| 保留周期 | 保留最近30天的备份文件 |
| 存储位置 | 异地存储或云存储 |
| 验证方式 | 定期恢复测试，确保备份可用 |

### 8.2 性能优化

**VACUUM - 数据库压缩：**

```sql
-- 回收空闲空间，优化文件大小
VACUUM;
```

> **使用场景：** 大量删除数据后执行，重建数据库文件以减少碎片。执行期间会锁定数据库，建议在低峰期操作。

**ANALYZE - 更新统计信息：**

```sql
-- 更新查询优化器的统计信息
ANALYZE;
```

> **使用场景：** 大量数据变更后执行，帮助查询优化器选择最优执行计划。

**REINDEX - 重建索引：**

```sql
-- 重建所有索引
REINDEX;

-- 重建指定索引
REINDEX idx_participants_status;
```

> **使用场景：** 索引碎片化严重或查询性能下降时执行。

**维护计划：**

| 操作 | 频率 | 说明 |
|------|------|------|
| ANALYZE | 每周 | 更新统计信息 |
| REINDEX | 每月 | 重建索引 |
| VACUUM | 每季度或数据大量删除后 | 压缩数据库文件 |

### 8.3 数据迁移

SQLite 对 `ALTER TABLE` 的支持有限，以下为常用迁移操作示例。

**添加新列：**

```sql
-- 添加新字段（SQLite支持）
ALTER TABLE participants ADD COLUMN department TEXT DEFAULT '';

ALTER TABLE participants ADD COLUMN remark TEXT DEFAULT '';
```

**重命名表（间接重命名列）：**

```sql
-- 1. 创建新表
CREATE TABLE participants_new (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT DEFAULT '',
    phone TEXT DEFAULT '',
    avatar TEXT DEFAULT '',
    full_name TEXT DEFAULT '',       -- 新增字段
    status TEXT DEFAULT '未中奖' CHECK(status IN ('未中奖', '已中奖')),
    is_blacklisted INTEGER DEFAULT 0 CHECK(is_blacklisted IN (0, 1)),
    weight INTEGER DEFAULT 1 CHECK(weight >= 0),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 2. 迁移数据
INSERT INTO participants_new (id, name, email, phone, avatar, status, is_blacklisted, weight, created_at)
SELECT id, name, email, phone, avatar, status, is_blacklisted, weight, created_at
FROM participants;

-- 3. 删除旧表
DROP TABLE participants;

-- 4. 重命名新表
ALTER TABLE participants_new RENAME TO participants;

-- 5. 重建索引和触发器
CREATE INDEX idx_participants_status ON participants(status);
CREATE INDEX idx_participants_blacklisted ON participants(is_blacklisted);
CREATE UNIQUE INDEX idx_participants_name_email ON participants(name, email);
```

**删除列（SQLite 3.35.0+）：**

```sql
-- SQLite 3.35.0+ 支持直接删除列
ALTER TABLE participants DROP COLUMN remark;
```

> **注意：** 对于 SQLite 3.35.0 以下版本，删除列需采用"创建新表 -> 迁移数据 -> 删除旧表 -> 重命名"的方式。

**迁移注意事项：**

| 注意事项 | 说明 |
|----------|------|
| 数据备份 | 迁移前务必备份数据库文件 |
| 外键约束 | 迁移时需注意外键依赖关系，按正确顺序操作 |
| 触发器 | 删除表后关联触发器也会被删除，需重新创建 |
| 索引 | 新表创建后需重新创建索引 |
| 事务保护 | 建议将迁移操作包裹在事务中，失败时可回滚 |
