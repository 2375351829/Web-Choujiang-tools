# 抽奖系统 部署方案
**版本**: v1.0.0 | **日期**: 2026-04-03

## 1. 部署概述

### 1.1 系统组成

| 组件 | 技术栈 | 开发端口 | 说明 |
|------|--------|----------|------|
| 后端API | Express.js + Socket.io | 3000 | REST API + WebSocket |
| 管理后台 | Vue 3 + Element Plus | 5173 | 管理员操作界面 |
| 抽奖展示 | Vue 3 + CSS3 Animations | 5174 | 大屏展示界面 |

### 1.2 部署模式

| 模式 | 说明 | 适用场景 |
|------|--------|----------|
| 开发模式 | 三端分别启动开发服务器 | 开发调试 |
| 生产模式 | 后端服务 + 前端构建静态文件 | 正式使用 |
| Docker模式 | 容器化部署 | 标准化部署 |

### 1.3 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| Node.js | 18.x | 20.x LTS |
| npm | 9.x | 10.x |
| 内存 | 512MB | 1GB+ |
| 磁盘 | 1GB | 2GB+ |
| 操作系统 | Windows 10 / Ubuntu 20.04 | Windows 11 / Ubuntu 22.04 |
| 浏览器 | Chrome 80+ | Chrome/Edge 最新版 |

## 2. 开发环境部署

### 2.1 环境准备

```bash
# 检查Node.js版本
node --version  # >= 18.x
npm --version   # >= 9.x

# 克隆项目
git clone <repository-url>
cd lottery-system
```

### 2.2 后端服务启动

```bash
cd server

npm run dev
# 服务运行在 http://localhost:3000
```


## 3. 生产环境部署

### 3.1 前端构建

```bash
# 构建管理后台
cd admin
npm run build
# 产物在 admin/dist/

# 构建抽奖展示
cd display
npm run build
# 产物在 display/dist/
```

### 3.2 后端服务配置

后端server.js需要同时托管两个前端构建产物：

```javascript
// 托管管理后台静态文件
app.use('/admin', express.static(path.join(__dirname, '../admin/dist')))
// 托管抽奖展示静态文件
app.use('/display', express.static(path.join(__dirname, '../display/dist')))
// 管理后台SPA fallback
app.get('/admin/*', (req, res) => {
  res.sendFile(path.join(__dirname, '../admin/dist/index.html'))
})
// 抽奖展示SPA fallback
app.get('/display/*', (req, res) => {
  res.sendFile(path.join(__dirname, '../display/dist/index.html'))
})
```

### 3.3 环境变量配置

创建 .env 文件：

```bash
NODE_ENV=production
PORT=3000
HOST=0.0.0.0
CORS_ORIGINS=http://localhost:3000,http://your-domain.com
UPLOAD_DIR=./uploads
DB_PATH=./data/lottery.db
```

### 3.4 PM2进程管理

```bash
# 安装PM2
npm install -g pm2

# 创建 ecosystem.config.js
module.exports = {
  apps: [{
    name: 'lottery-server',
    script: 'server/server.js',
    cwd: '/path/to/lottery-system',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    instances: 1,
    autorestart: true,
    max_memory_restart: '500M',
    log_date_format: 'YYYY-MM-DD HH:mm:ss'
  }]
}

# 启动
pm2 start ecosystem.config.js

# 常用命令
pm2 list          # 查看进程
pm2 logs          # 查看日志
pm2 restart all   # 重启
pm2 stop all      # 停止
pm2 save          # 保存进程列表
pm2 startup       # 设置开机自启
```

## 4. Nginx反向代理配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 管理后台
    location /admin {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 抽奖展示
    location /display {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # API
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Socket.io
    location /socket.io {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 上传文件
    location /uploads {
        proxy_pass http://127.0.0.1:3000;
    }

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1000;
}
```

## 5. Docker部署

### 5.1 Dockerfile

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN mkdir -p admin/dist display/dist
RUN cd admin && npm ci && npm run build
RUN cd display && npm ci && npm run build

FROM node:20-alpine
WORKDIR /app
COPY --from=builder /app/server ./server
COPY --from=builder /app/admin/dist ./admin/dist
COPY --from=builder /app/display/dist ./display/dist
COPY --from=builder /app/server/package*.json ./server/
RUN cd server && npm ci --only=production
RUN mkdir -p /app/uploads /app/data
EXPOSE 3000
CMD ["node", "server/server.js"]
```

### 5.2 docker-compose.yml

```yaml
version: '3.8'
services:
  lottery:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - ./data:/app/data
      - ./uploads:/app/uploads
    environment:
      - NODE_ENV=production
      - PORT=3000
    restart: always
```

### 5.3 Docker命令

```bash
# 构建并启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f

# 停止
docker-compose down

# 重启
docker-compose restart
```

## 6. 运维方案

### 6.1 日志管理

- PM2日志：pm2 logs
- 应用日志：按日期轮转，保留30天
- 访问日志：记录API请求

### 6.2 数据库备份

```bash
# 定时备份脚本 (crontab)
# 每天凌晨2点备份
0 2 * * * cp /path/to/data/lottery.db /path/to/backups/lottery_$(date +\%Y\%m\%d).db
# 保留最近30天备份
0 3 * * * find /path/to/backups -name "lottery_*.db" -mtime +30 -delete
```

### 6.3 监控方案

- 进程监控：PM2自动重启
- 磁盘监控：定期检查磁盘空间
- 端口监控：检查3000端口是否在监听

### 6.4 更新部署流程

```bash
# 1. 备份数据
cp data/lottery.db data/lottery_backup.db

# 2. 拉取代码
git pull

# 3. 安装依赖
cd server && npm install
cd admin && npm install
cd display && npm install

# 4. 构建
cd admin && npm run build
cd display && npm run build

# 5. 重启服务
pm2 restart lottery-server
```

## 7. 安全加固

### 7.1 网络安全

- 仅开放必要端口（80/443）
- 配置防火墙规则
- 使用HTTPS（Let's Encrypt）

### 7.2 应用安全

- CORS白名单配置
- API频率限制
- 文件上传安全
- 环境变量管理敏感信息

### 7.3 数据安全

- 数据库文件权限设置
- 定期备份
- 日志脱敏
