# 抽奖系统 系统架构设计
**版本**: v1.0.0 | **日期**: 2026-04-03

## 1. 架构概述

### 1.1 设计原则

- **前后端分离**：管理后台、抽奖展示、后端API三者独立
- **单一职责**：每个模块只负责一个功能领域
- **高内聚低耦合**：模块间通过API和WebSocket通信
- **可扩展性**：支持水平扩展和功能扩展

### 1.2 架构风格

采用前后端分离的B/S架构，结合WebSocket实时通信。

## 5. 核心算法设计

### 5.1 加权随机抽奖算法

```
算法名称：加权随机选择 (Weighted Random Selection)
时间复杂度：O(n)，n为参与者数量
空间复杂度：O(n)

输入：参与者列表（含权重）、已中奖者列表、黑名单列表
输出：中奖者

步骤：
1. 过滤：排除已中奖者和黑名单用户
2. 边界检查：若无可用参与者，返回错误
3. 构建加权列表：使用参与者自身权重
4. 计算总权重：totalWeight = Σ(weight_i)
5. 生成随机数：random = Math.random() * totalWeight
6. 遍历选择：累减权重，random <= 0时选中
7. 浮点精度保底：返回最后一个参与者

概率公式：P(参与者i) = weight_i / Σ(weight_j)
```

### 5.2 抽奖状态机

```
状态：IDLE -> DRAWING -> PAUSED -> STOPPED -> IDLE

转换规则：
- IDLE + start() -> DRAWING
- DRAWING + pause() -> PAUSED
- PAUSED + resume() -> DRAWING
- DRAWING + stop() -> STOPPED
- PAUSED + stop() -> STOPPED
- STOPPED + reset() -> IDLE
- IDLE + reset() -> IDLE
- DRAWING + reset() -> IDLE
- PAUSED + reset() -> IDLE
```

## 6. 通信设计

### 6.1 管理后台 -> 后端

- **HTTP REST API**：数据CRUD操作
- **WebSocket**：抽奖控制指令

### 6.2 抽奖展示 -> 后端

- **HTTP REST API**：获取配置和初始数据
- **WebSocket**：接收抽奖事件、报告状态

### 6.3 后端 -> 抽奖展示

- **WebSocket**：广播抽奖事件（开始/暂停/停止/结果/重置）

### 6.4 通信时序

```
管理员点击开始 -> Admin发送POST /api/console/start
  -> Server执行抽奖算法
  -> Server广播 draw:start (via WebSocket)
  -> Display收到事件，开始动画
  -> 管理员点击停止 -> Admin发送POST /api/console/stop
  -> Server广播 draw:stop + draw:result (via WebSocket)
  -> Display收到事件，停止动画，展示结果
```

## 7. 安全设计

### 7.1 输入验证

- 所有API参数进行类型和范围验证
- 状态值使用白名单验证
- SQL使用参数化查询

### 7.2 文件上传安全

- 文件类型白名单
- 文件大小限制（10MB）
- 文件名随机化

### 7.3 CORS配置

- 限制允许的来源域名
- 仅允许必要的方法和头部

### 7.4 频率限制

- API接口频率限制
- 防止恶意请求

## 8. 性能设计

### 8.1 前端性能

- Vite代码分割和懒加载
- CSS动画使用transform/opacity（GPU加速）
- 组件按需加载

### 8.2 后端性能

- 数据库索引优化
- 静态文件缓存
- Gzip压缩

### 8.3 WebSocket性能

- 事件压缩：状态变化时才推送
- 心跳检测保持连接
- 断线自动重连
