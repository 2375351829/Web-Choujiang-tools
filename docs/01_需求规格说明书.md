# 抽奖系统 需求规格说明书
**版本**: v1.0.0 | **日期**: 2026-04-03

## 1. 引言
### 1.1 编写目的
本文档详细描述抽奖系统的功能需求和非功能需求，作为系统设计、开发、测试的依据。

### 1.2 系统范围
系统包含两个独立前端应用（管理后台、抽奖展示）和一个后端API服务。管理后台负责数据管理和抽奖控制，抽奖展示负责大屏动画展示。

### 1.3 术语定义
| 术语 | 定义 |
| 场次(Session) | 一次抽奖活动的容器，包含参与者和奖品 |
| 参与者(Participant) | 参与抽奖的人员 |
| 奖品(Prize) | 抽奖活动的奖品 |
| 加权随机 | 根据权重值差异化概率的随机选择算法 |
| 黑名单 | 被排除在抽奖之外的人员列表 |

## 2. 功能需求
### 2.1 人员管理模块
#### FR-P-001 人员创建
- 优先级：高
- 描述：管理员可以手动添加参与者
- 输入：姓名(必填)、邮箱(选填)、电话(选填)、中奖权重(默认1)
- 输出：创建成功/失败提示
- 业务规则：姓名+邮箱组合唯一，不可重复创建

#### FR-P-002 人员编辑
- 优先级：高
- 描述：管理员可以修改参与者信息
- 输入：参与者ID、修改字段
- 输出：更新成功/失败提示

#### FR-P-003 人员删除
- 优先级：高
- 描述：管理员可以删除参与者
- 输入：参与者ID
- 输出：删除成功/失败提示
- 业务规则：删除后关联的抽奖记录保留

#### FR-P-004 批量导入
- 优先级：高
- 描述：支持通过Excel文件批量导入参与者
- 输入：Excel文件(.xlsx/.xls)、可选场次ID
- 输出：导入结果（成功数、跳过数）
- 业务规则：Excel必须包含name列，email和phone列可选

#### FR-P-005 黑名单管理
- 优先级：中
- 描述：管理员可以将参与者加入/移出黑名单
- 输入：参与者ID、黑名单状态(0/1)
- 输出：状态更新成功
- 业务规则：黑名单用户不参与抽奖

#### FR-P-006 人员查询
- 优先级：中
- 描述：按状态、黑名单、关键词筛选参与者
- 输入：筛选条件
- 输出：参与者列表

### 2.2 奖品管理模块
#### FR-Z-001 奖品创建
- 优先级：高
- 描述：管理员可以创建奖品
- 输入：奖品名称(必填)、描述(选填)、图片(选填)、可抽次数(默认1)
- 输出：创建成功/失败提示

#### FR-Z-002 奖品编辑/删除
- 优先级：高
- 描述：管理员可以修改或删除奖品信息

#### FR-Z-003 奖品图片上传
- 优先级：低
- 描述：支持上传奖品图片
- 业务规则：支持jpg/png/gif，最大10MB

### 2.3 场次管理模块
#### FR-S-001 场次创建
- 优先级：高
- 描述：创建抽奖场次
- 输入：场次名称(必填)、描述(选填)
- 输出：场次信息

#### FR-S-002 场次配置
- 优先级：高
- 描述：为场次关联参与者和奖品
- 业务规则：参与者和奖品均为多对多关系

#### FR-S-003 场次状态管理
- 优先级：高
- 描述：管理场次生命周期（未开始/进行中/已结束）

### 2.4 抽奖控制模块
#### FR-L-001 开始抽奖
- 优先级：高
- 描述：管理员触发抽奖开始
- 输入：场次ID、奖品ID
- 输出：展示端开始播放动画
- 业务规则：通过WebSocket广播draw:start事件

#### FR-L-002 暂停抽奖
- 优先级：中
- 描述：暂停抽奖动画
- 业务规则：暂停后可恢复

#### FR-L-003 停止抽奖
- 优先级：高
- 描述：停止动画并执行抽奖算法
- 输出：中奖结果
- 业务规则：后端执行加权随机算法，结果通过WebSocket广播

#### FR-L-004 重置抽奖
- 优先级：中
- 描述：重置抽奖状态，准备下一轮

#### FR-L-005 加权随机算法
- 优先级：高
- 描述：根据参与者权重执行加权随机选择
- 算法：P(i) = weight_i / Σ(weight_j)
- 业务规则：
  - 权重为0的参与者不会被选中
  - 已中奖同一奖品的参与者被排除
  - 黑名单用户被排除
  - 奖品库存不足时阻止抽奖

### 2.5 展示模块
#### FR-D-001 动画展示
- 优先级：高
- 描述：抽奖时播放动画效果
- 支持三种模式：Grid(网格)、Sphere(球体)、List(列表)

#### FR-D-002 中奖展示
- 优先级：高
- 描述：中奖时展示中奖者信息和奖品信息
- 效果：彩带飘落、姓名放大、奖品展示

#### FR-D-003 实时同步
- 优先级：高
- 描述：展示端与管理端实时同步
- 技术：Socket.io WebSocket

#### FR-D-004 连接状态
- 优先级：中
- 描述：展示端显示与服务器连接状态

### 2.6 系统配置模块
#### FR-C-001 主题配置
- 优先级：低
- 描述：支持6种预设主题切换

#### FR-C-002 背景配置
- 优先级：低
- 描述：支持渐变/纯色/图片三种背景类型

#### FR-C-003 动画配置
- 优先级：低
- 描述：配置动画类型和轮播速度

## 3. 非功能需求
### 3.1 性能需求
| 指标 | 要求 |
| 页面加载时间 | < 2秒 |
| API响应时间 | < 100毫秒 |
| WebSocket延迟 | < 50毫秒 |
| 并发支持 | 100+ 同时在线 |
| 动画帧率 | >= 60fps |

### 3.2 安全需求
- SQL注入防护：使用参数化查询
- 文件上传安全：类型白名单、大小限制
- 输入验证：所有API输入进行校验
- CORS配置：限制允许的来源

### 3.3 可用性需求
- 响应式设计：适配手机/平板/桌面/大屏
- 浏览器兼容：Chrome 80+、Edge 80+、Firefox 80+
- 操作简便：核心操作不超过3步

### 3.4 可维护性需求
- 前后端分离：独立开发部署
- 代码规范：统一的代码风格
- 日志记录：关键操作日志
- 数据备份：支持数据库备份恢复

### 3.5 可扩展性需求
- 模块化设计：便于功能扩展
- 插件化动画：便于添加新动画效果
- API版本化：便于接口升级

## 4. 用例分析
### 4.1 管理员用例
#### UC-001 完整抽奖流程
- 参与者：管理员
- 前置条件：系统已启动
- 主流程：
  1. 管理员登录管理后台
  2. 导入参与者名单
  3. 创建奖品
  4. 创建抽奖场次
  5. 为场次关联参与者和奖品
  6. 在控制台选择场次和奖品
  7. 点击开始抽奖
  8. 等待动画播放
  9. 点击停止抽奖
  10. 查看中奖结果
  11. 重置后进行下一轮
- 后置条件：中奖结果已记录

#### UC-002 批量导入参与者
- 参与者：管理员
- 前置条件：已准备Excel文件
- 主流程：
  1. 进入人员管理页面
  2. 点击批量导入
  3. 选择Excel文件
  4. 系统解析并导入
  5. 显示导入结果

### 4.2 展示端用例
#### UC-003 观看抽奖
- 参与者：观众（通过大屏）
- 前置条件：展示页面已打开并连接服务器
- 主流程：
  1. 展示页面自动加载配置
  2. 显示活动标题和参与者轮播
  3. 收到抽奖开始事件
  4. 播放抽奖动画
  5. 收到抽奖结果事件
  6. 展示中奖者信息
  7. 播放中奖特效

## 5. 约束条件
### 5.1 技术约束
- 后端必须使用Node.js
- 前端必须使用Vue 3
- 数据库使用SQLite
- 实时通信使用Socket.io

### 5.2 业务约束
- 同一参与者不能重复中同一奖品
- 黑名单用户不参与抽奖
- 抽奖结果不可篡改（需审计追踪）
