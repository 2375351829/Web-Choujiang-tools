## 2. 人员管理模块

### 2.1 功能概述

人员管理模块负责管理参与抽奖的人员信息，支持单个添加和批量导入两种方式。

![人员管理页面](./screenshots/admin-personnel.png)

### 2.2 功能详情

#### 2.2.1 人员创建 (FR-P-001)

**功能描述**：管理员可以手动添加参与者

| 项目 | 说明 |
|------|------|
| 输入 | 姓名(必填)、邮箱(选填)、电话(选填)、中奖权重(默认1) |
| 输出 | 创建成功/失败提示 |
| 业务规则 | 姓名+邮箱组合唯一，不可重复创建 |

**操作流程**：
1. 点击「添加人员」按钮
2. 填写人员信息表单
3. 点击「确定」提交
4. 系统验证并创建人员

#### 2.2.2 人员编辑 (FR-P-002)

**功能描述**：管理员可以修改参与者信息

| 项目 | 说明 |
|------|------|
| 输入 | 参与者ID、修改字段 |
| 输出 | 更新成功/失败提示 |

**可编辑字段**：
- 姓名
- 邮箱
- 电话
- 中奖权重

#### 2.2.3 人员删除 (FR-P-003)

**功能描述**：管理员可以删除参与者

| 项目 | 说明 |
|------|------|
| 输入 | 参与者ID |
| 输出 | 删除成功/失败提示 |
| 业务规则 | 删除后关联的抽奖记录保留 |

**注意**：删除操作不可恢复，请谨慎操作。

#### 2.2.4 批量导入 (FR-P-004)

**功能描述**：支持通过Excel文件批量导入参与者

| 项目 | 说明 |
|------|------|
| 输入 | Excel文件(.xlsx/.xls)、可选场次ID |
| 输出 | 导入结果（成功数、跳过数） |
| 业务规则 | Excel必须包含name列，email和phone列可选 |

**Excel格式要求**：

| 列名 | 必填 | 说明 |
|------|------|------|
| name | 是 | 参与者姓名 |
| email | 否 | 邮箱地址 |
| phone | 否 | 电话号码 |
| weight | 否 | 中奖权重，默认为1 |

**导入流程**：
1. 点击「批量导入」按钮
2. 选择Excel文件
3. 系统解析文件内容
4. 验证数据格式
5. 批量插入数据库
6. 显示导入结果

#### 2.2.5 黑名单管理 (FR-P-005)

**功能描述**：管理员可以将参与者加入/移出黑名单

| 项目 | 说明 |
|------|------|
| 输入 | 参与者ID、黑名单状态(0/1) |
| 输出 | 状态更新成功 |
| 业务规则 | 黑名单用户不参与抽奖 |

**使用场景**：
- 已离职员工
- 不符合参与条件的人员
- 其他需要排除的人员

#### 2.2.6 人员查询 (FR-P-006)

**功能描述**：按条件筛选参与者

**筛选条件**：
- 状态：全部/已中奖/未中奖
- 黑名单：全部/正常/黑名单
- 关键词：姓名/邮箱/电话模糊搜索

### 2.3 数据字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | 自动 | 主键ID |
| name | TEXT | 是 | 姓名 |
| email | TEXT | 否 | 邮箱 |
| phone | TEXT | 否 | 电话 |
| weight | INTEGER | 否 | 中奖权重，默认1 |
| is_blacklisted | INTEGER | 否 | 黑名单状态，默认0 |
| created_at | DATETIME | 自动 | 创建时间 |

![场次管理页面](./screenshots/admin-sessions.png)

### 4.2 功能详情

**功能描述**：创建抽奖场次

**场次属性**：

| 属性 | 类型 | 必填 | 说明 |

#### 4.2.2 场次配置 (FR-S-002)

## 5. 抽奖控制模块

### 5.1 功能概述

抽奖控制模块是系统的核心功能，负责控制抽奖流程和执行抽奖算法。

![控制台页面](./screenshots/admin-dashboard.png)

### 5.2 功能详情

#### 5.2.1 开始抽奖 (FR-L-001)

**功能描述**：管理员触发抽奖开始

| 项目 | 说明 |
|------|------|
| 输入 | 场次ID、奖品ID |
| 输出 | 展示端开始播放动画 |
| 业务规则 | 通过WebSocket广播draw:start事件 |

**前置条件**：
- 已选择场次和奖品
- 场次已关联参与者
- 奖品库存充足
- 展示端已连接

#### 5.2.2 暂停抽奖 (FR-L-002)

**功能描述**：暂停抽奖动画

| 项目 | 说明 |
|------|------|
| 业务规则 | 暂停后可恢复 |

**使用场景**：
- 需要临时中断抽奖
- 配合活动节奏

#### 5.2.3 停止抽奖 (FR-L-003)

**功能描述**：停止动画并执行抽奖算法

| 项目 | 说明 |
|------|------|
| 输出 | 中奖结果 |
| 业务规则 | 后端执行加权随机算法，结果通过WebSocket广播 |

**执行流程**：
1. 停止动画
2. 执行抽奖算法
3. 记录中奖结果
4. 更新奖品库存
5. 广播中奖信息

#### 5.2.4 重置抽奖 (FR-L-004)

**功能描述**：重置抽奖状态，准备下一轮

**重置操作**：
- 清除当前抽奖状态
- 重置展示端动画
- 准备下一轮抽奖

#### 5.2.5 加权随机算法 (FR-L-005)

**功能描述**：根据参与者权重执行加权随机选择

**算法说明**：

```
算法名称：加权随机选择 (Weighted Random Selection)
时间复杂度：O(n)，n为参与者数量
空间复杂度：O(n)

输入：参与者列表（含权重）、已中奖者列表、黑名单列表
输出：中奖者

步骤：
1. 过滤：排除已中奖者和黑名单用户
2. 边界检查：若无可用参与者，返回错误
3. 构建加权列表：使用参与者自身权重
4. 计算总权重：totalWeight = Σ(weight_i)
5. 生成随机数：random = Math.random() * totalWeight
6. 遍历选择：累减权重，random <= 0时选中
7. 浮点精度保底：返回最后一个参与者

概率公式：P(参与者i) = weight_i / Σ(weight_j)
```

**业务规则**：
- 权重为0的参与者不会被选中
- 已中奖同一奖品的参与者被排除
- 黑名单用户被排除
- 奖品库存不足时阻止抽奖

### 5.3 状态机

抽奖过程遵循状态机模式：

```
状态：IDLE -> DRAWING -> PAUSED -> STOPPED -> IDLE

转换规则：
- IDLE + start() -> DRAWING
- DRAWING + pause() -> PAUSED
- PAUSED + resume() -> DRAWING
- DRAWING + stop() -> STOPPED
- PAUSED + stop() -> STOPPED
- STOPPED + reset() -> IDLE
- IDLE + reset() -> IDLE
- DRAWING + reset() -> IDLE
- PAUSED + reset() -> IDLE
```

### 5.4 按钮状态控制

| 状态 | 开始 | 暂停 | 停止 | 重置 |
|------|------|------|------|------|
| IDLE | 可用 | 禁用 | 禁用 | 可用 |
| DRAWING | 禁用 | 可用 | 可用 | 可用 |
| PAUSED | 禁用 | 禁用 | 可用 | 可用 |
| STOPPED | 禁用 | 禁用 | 禁用 | 可用 |

## 6. 展示模块

### 6.1 功能概述

展示模块是面向观众的大屏展示界面，用于实时展示抽奖动画和中奖结果。

![展示端主界面](./screenshots/display-main.png)

### 6.2 功能详情

#### 6.2.1 动画展示 (FR-D-001)

**功能描述**：抽奖时播放动画效果

**支持三种模式**：

| 模式 | 效果描述 | 适用场景 |
|------|----------|----------|
| Grid（网格） | 3x3网格布局，高亮格子快速跳动 | 参与者较少时 |
| Sphere（球体） | 3D球体效果，参与者姓名分布在球面旋转 | 参与者较多时 |
| List（列表） | 纵向列表展示，高亮项快速滚动 | 需要清晰展示时 |

**动画效果**：

**Grid模式**：
- 3x3网格布局，每格显示一个参与者姓名
- 当前高亮格子发光效果（box-shadow + scale）
- 高亮在格子间快速跳动
- 中奖时：目标格子放大 + 金色边框 + 其他格子淡出

**Sphere模式**：
- 3D球体效果，参与者姓名分布在球面
- 球体持续旋转
- 当前高亮姓名放大 + 发光
- 中奖时：球体停止旋转，中奖者姓名居中放大

**List模式**：
- 纵向列表展示参与者
- 当前高亮项背景色变化 + 左侧指示器
- 高亮在列表中快速滚动
- 中奖时：列表停止，中奖项居中 + 特殊样式

#### 6.2.2 中奖展示 (FR-D-002)

**功能描述**：中奖时展示中奖者信息和奖品信息

**展示效果**：
- 彩带飘落效果（canvas-confetti）
- 中奖者姓名弹出放大动画
- 奖品名称渐入显示
- 背景闪光效果

#### 6.2.3 实时同步 (FR-D-003)

**功能描述**：展示端与管理端实时同步

| 项目 | 说明 |
|------|------|
| 技术 | Socket.io WebSocket |
| 事件 | draw:start, draw:stop, draw:result, draw:reset |
| 重连 | 自动重连机制 |

**通信时序**：

```
管理员点击开始 -> Admin发送POST /api/console/start
  -> Server执行抽奖算法
  -> Server广播 draw:start (via WebSocket)
  -> Display收到事件，开始动画
  -> 管理员点击停止 -> Admin发送POST /api/console/stop
  -> Server广播 draw:stop + draw:result (via WebSocket)
  -> Display收到事件，停止动画，展示结果
```

#### 6.2.4 连接状态 (FR-D-004)

**功能描述**：展示端显示与服务器连接状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| 已连接 | 绿色 | 与服务器正常连接 |
| 连接中 | 黄色 | 正在尝试连接 |
| 断开连接 | 红色 | 与服务器断开 |

### 6.3 响应式设计

**断点设置**：

| 断点 | 宽度 | 字体基准 | 网格大小 |
|------|------|----------|----------|
| mobile | <640px | 14px | 2x2 |
| tablet | 640-1024px | 18px | 3x3 |
| desktop | 1024-1920px | 24px | 3x3 |
| large | >1920px | 28px | 4x4 |

## 7. 系统配置模块

### 7.1 功能概述

系统配置模块用于设置展示端的显示效果，包括主题、背景、动画等配置。

![系统配置页面](./screenshots/admin-config.png)

### 7.2 功能详情

#### 7.2.1 主题配置 (FR-C-001)

**功能描述**：支持6种预设主题切换

| 主题 | 背景色 | 主色调 | 强调色 |
|------|--------|--------|--------|
| 暗夜黑 | #0f172a | #1e293b | #3b82f6 |
| 简约白 | #ffffff | #f8fafc | #3b82f6 |
| 海洋蓝 | #0c4a6e | #075985 | #06b6d4 |
| 森林绿 | #14532d | #166534 | #22c55e |
| 落日橙 | #7c2d12 | #9a3412 | #f97316 |
| 梦幻紫 | #3b0764 | #581c87 | #a855f7 |

#### 7.2.2 背景配置 (FR-C-002)

**功能描述**：支持渐变/纯色/图片三种背景类型

| 类型 | 配置项 |
|------|--------|
| 渐变 | 起始颜色、结束颜色、渐变方向 |
| 纯色 | 背景颜色 |
| 图片 | 图片文件上传 |

#### 7.2.3 动画配置 (FR-C-003)

**功能描述**：配置动画类型和轮播速度

| 配置项 | 说明 |
|--------|------|
| 动画类型 | Grid/Sphere/List |
| 轮播速度 | 参与者姓名轮播速度 |

## 8. 技术特性

### 8.1 性能指标

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2秒 |
| API响应时间 | < 100毫秒 |
| WebSocket延迟 | < 50毫秒 |
| 并发支持 | 100+ 同时在线 |
| 动画帧率 | >= 60fps |

### 8.2 安全特性

| 特性 | 说明 |
|------|------|
| SQL注入防护 | 使用参数化查询 |
| 文件上传安全 | 类型白名单、大小限制 |
| 输入验证 | 所有API输入进行校验 |
| CORS配置 | 限制允许的来源 |

### 8.3 技术栈

**后端**：
- Express.js 4.x - Web框架
- Socket.io 4.x - WebSocket
- better-sqlite3 9.x - SQLite驱动
- Multer 1.x - 文件上传

**管理后台**：
- Vue.js 3.x - 前端框架
- Element Plus - UI组件库
- Pinia - 状态管理
- Axios - HTTP客户端

**展示端**：
- Vue.js 3.x - 前端框架
- Socket.io-client - WebSocket客户端
- canvas-confetti - 彩带效果
